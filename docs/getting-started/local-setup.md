# Local Development Setup

Complete guide for setting up ValueOS for local development.

## System Requirements

### Hardware

- **RAM**: 8GB minimum, 16GB recommended
- **Disk**: 10GB free space for Docker images and data
- **CPU**: 4+ cores recommended

### Software

- **Docker Desktop**: v4.0+ with Docker Compose V2
- **Node.js**: v20+ (use `.nvmrc`)
- **Corepack**: required for pnpm
- **Git**: v2.30+
- **IDE**: VS Code recommended with extensions:
  - ESLint
  - Prettier
  - Tailwind CSS IntelliSense

## Local Dev Quickstart (Default)

These commands match the canonical quickstart and are copy/paste runnable.

```bash
git clone https://github.com/valynt/valueos.git
cd valueos
corepack enable
corepack prepare pnpm@9.15.0 --activate
pnpm install
pnpm run dx:env -- --mode local --force
pnpm run dx
```

**Expected outcome:** `.env.local` and `.env.ports` are generated, Docker dependencies start, Supabase runs locally, and Vite serves the app at http://localhost:5173.

## Development Modes

### Option A (Default): Local mode (host + Docker deps)

Use this when you want fast HMR with Docker only for dependencies (Postgres, Redis, Supabase).

```bash
pnpm run dx:env -- --mode local --force
pnpm run dx
```

**Expected outcome:** frontend/backend run on the host, dependencies run in Docker.

### Option B: Full Docker mode

Use this when you want everything inside Docker containers.

```bash
pnpm run dx:env -- --mode docker --force
pnpm run dx:docker
```

**Expected outcome:** frontend, backend, and dependencies run in Docker with Caddy routing.

## Environment Files

All environment files are generated by `scripts/dx/env-compiler.js`.

| File | Purpose |
| --- | --- |
| `.env.local` | Main environment file (local or docker mode) |
| `.env.ports` | Port mappings for Docker Compose |
| `deploy/envs/.env.ports` | Docker Compose env file for containers |

## Day-to-day Commands

### Start or stop the stack

```bash
pnpm run dx          # Start (local mode)
pnpm run dx:docker   # Start (docker mode)
pnpm run dx:down     # Stop containers
pnpm run dx:check    # Health check all services
pnpm run dx:logs     # Tail logs
```

### Database operations

```bash
pnpm run db:reset   # Reset database + apply migrations
pnpm run db:push    # Apply migrations without full reset
pnpm run db:types   # Generate TypeScript types from local DB
```

### Seed demo data

```bash
pnpm run seed:demo
```

### Frontend or backend only

```bash
pnpm run dev         # Frontend only (ValyntApp)
pnpm run backend:dev # Backend only
```

## Service URLs

When running in local mode (`pnpm run dx`):

- **Frontend**: `http://localhost:5173`
- **Backend API**: `http://localhost:3001`
- **Backend Health**: `http://localhost:3001/health`
- **Supabase API**: `http://localhost:54321`
- **Supabase Studio**: `http://localhost:54323`
- **Supabase DB**: `localhost:54322`
- **PostgreSQL (deps container)**: `localhost:5432`
- **Redis**: `localhost:6379`
- **Caddy Admin** (docker mode only): `http://localhost:2019`

## Optional: Caddy reverse proxy (docker mode)

```bash
pnpm run dx:caddy:start
pnpm run dx:caddy:logs
pnpm run dx:caddy:stop
```

## IDE Configuration

### VS Code Settings

Create `.vscode/settings.json`:

```json
{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": true
  },
  "typescript.preferences.importModuleSpecifier": "relative"
}
```
