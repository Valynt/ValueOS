# ValueOS Dev Reliability Spec & Hardening Plan

## A) Dev Reliability Spec (Standard)

**Supported OS targets**
- macOS (Intel + Apple Silicon)
- Windows 11 + WSL2 (Ubuntu 22.04+)
- Linux (Ubuntu 22.04+, Debian 12+)

**Required versions**
- Docker Desktop / Engine: 24.x+
- Docker Compose: v2.x (bundled with Docker Desktop)
- Node.js: **20.19.0** (pinned via `.nvmrc`)
- pnpm: **9.15.0** (via corepack)

**Deterministic commands**
- Bootstrap: `./dev up` (or `pnpm dx up`)
- Doctor: `./dev doctor`
- Logs: `./dev logs <service>`
- Reset: `./dev reset`
- Down: `./dev down`

**Deterministic installs**
- Single package manager: **pnpm**
- Lockfile enforced locally and in CI
- Offline-cache-friendly: `pnpm install --frozen-lockfile --prefer-offline`

## C) Root Causes of Flakiness â†’ Fixes

1. **Package manager drift (npm vs pnpm)**
   - **Fix:** Standardize on pnpm, enforce via corepack + CI `--frozen-lockfile`.

2. **Docker env mismatch + missing env files**
   - **Fix:** Compose now reads root `.env.ports`/`.env.local`; `./dev up` always regenerates env.

3. **Backend/frontend Dockerfiles not aligned to monorepo layout**
   - **Fix:** Dockerfiles updated to build/run from `apps/ValyntApp` and `packages/backend`.

4. **Port conflicts on first boot**
   - **Fix:** `./dev up` checks for conflicts and fails with actionable fixes.

5. **Service boot race conditions**
   - **Fix:** Healthchecks + `depends_on` health gating for all services.

## D) Failure Playbook (Top 10)

1. **Port collision on 5173 or 3001**  
   - Fix: `export VITE_PORT=5175` or `export API_PORT=3003`  
   - Regenerate env: `pnpm run dx:env --mode local --force`  
   - Retry: `./dev up`

2. **Node version mismatch**  
   - Fix: `nvm install && nvm use` (uses `.nvmrc`)

3. **Lockfile drift / install failure**  
   - Fix: `pnpm install --frozen-lockfile --prefer-offline`  
   - If lockfile changed, commit updated `pnpm-lock.yaml`.

4. **Docker daemon not running**  
   - Fix: Start Docker Desktop / `systemctl start docker`  
   - Retry: `./dev up`

5. **Supabase not running**  
   - Fix: `supabase start` (or let `./dev up` handle it)  
   - Verify: `curl http://localhost:54321/rest/v1/`

6. **Backend health check failing**  
   - Fix: `./dev logs backend`  
   - Check DB: `docker compose ps postgres`  
   - Retry: `./dev reset`

7. **Frontend not reachable**  
   - Fix: `./dev logs frontend`  
   - Verify nginx: `curl http://localhost:5173/health`

8. **Redis connection errors**  
   - Fix: `docker compose ps redis`  
   - Retry: `./dev reset`

9. **Kafka / schema registry errors**  
   - Fix: `./dev logs kafka` / `./dev logs schema-registry`  
   - Retry: `./dev reset`

10. **Corrupted Docker volumes / caches**  
   - Fix: `./dev reset`  
   - Optional: `pnpm run dx:clean`
