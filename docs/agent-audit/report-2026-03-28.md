### 3. Critical Issues Found

#### Issue 1: Heavy Mocking in Tests
**Location:** All `__tests__/*.test.ts` files
**Problem:** Tests use `vi.mock()` to mock LLMGateway, MemorySystem, CircuitBreaker, and other dependencies. This means tests verify the mocks, not the actual integration.

**Example from OpportunityAgent.test.ts:**
```typescript
const VALID_LLM_RESPONSE = JSON.stringify({
  hypotheses: [{
    id: 'acme-rev-1',  // HARDCODED FAKE DATA
    title: 'Increase ACME Corp Revenue by Expanding into APAC',
    category: 'revenue_growth',
    estimated_impact: { low: 500000, high: 1200000, unit: 'USD' },
    confidence: 0.85,
    evidence: ['Market analysis shows 40% growth potential'],
    assumptions: ['Regional partnerships established within 6 months']
  }]
});
```

**Impact:** Tests pass even if the actual LLM integration is broken.

#### Issue 2: DiscoveryAgent Architecture
**Location:** `DiscoveryAgent.ts` lines 302-391
**Problem:** DiscoveryAgent creates fresh `OpportunityAgent`, `MemorySystem`, `LLMGateway`, and `CircuitBreaker` instances for each run (lines 309-317). This:
- Bypasses the agent factory
- Creates resource churn
- Makes testing harder
- Loses circuit breaker state

#### Issue 3: Empty Prompts in DiscoveryAgent
**Location:** `DiscoveryAgent.ts` lines 330-333
```typescript
prompts: {
  system_prompt: "",
  user_prompt_template: "",
},
```
**Problem:** OpportunityAgent is instantiated with empty prompts, relying entirely on internal defaults.
